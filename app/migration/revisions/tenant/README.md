## Introduction

Alembic is a database migration tool for SQLAlchemy. It helps you manage changes to the database schema, keeping your database in sync with changes in your application models. This project demonstrates how to set up and use Alembic for database migrations.

- [alembic==1.14.0 ](https://alembic.sqlalchemy.org/en/latest/)
- [SQLAlchemy==2.0.36](https://docs.sqlalchemy.org/en/20/orm/quickstart.html)

## Using Alembic
### Create a Migration
Whenever you change your application’s models (e.g., add a new table, modify an existing one), you need to create a new migration to reflect these changes in the database schema. To create a migration, run:

```bash
docker ps
docker exec -it <container name> sh
alembic revision --autogenerate -m "Describe the change"
```

- --autogenerate will compare the current database schema with your SQLAlchemy models and generate the necessary migration steps.
- -m "Describe the change" provides a brief description of the migration.
- After running the command, a new migration file will be created in the app/migrations/versions folder.

```py
"""add_table_test

Revision ID: 08e0921a7b30
Revises: 
Create Date: 2025-01-07 11:41:50.542014+09:00

"""
from alembic import op
import sqlalchemy as sa
from app.migrations.for_each_schema import for_each_schema


# revision identifiers, used by Alembic.
revision = '08e0921a7b30'
down_revision = None
branch_labels = None
depends_on = None

@for_each_schema
def upgrade(schema: str) -> None:
    # ### commands auto generated by Alembic - please adjust! ###
    op.create_table('test_migration',
    sa.Column('id', sa.String(length=255), nullable=False),
    sa.Column('name', sa.String(length=255), nullable=False),
    sa.PrimaryKeyConstraint('id', name=op.f('pk_test_migration')),
    schema=schema ### <- here !!!
    )
    # ### end Alembic commands ###


@for_each_schema
def downgrade(schema: str) -> None:
    # ### commands auto generated by Alembic - please adjust! ###
    op.drop_table('test_migration', schema=schema) ### <- here !!!
    # ### end Alembic commands ###

```

- Please change schema=schema

### Apply a Migration
To apply all pending migrations, use the following command:
```bash
alembic upgrade head
```
This will apply all migrations from the current database version to the latest (head)

### Rollback a Migration
If you need to revert a migration and rollback to a previous version, you can use:

```bash
alembic downgrade -1
```
- The -1 will rollback the last migration. You can replace -1 with a specific version ID to revert to a particular migration.
